OVERVIEW
--------
Healthy Food Bank is a full-stack web application that connects customers with
local vendors to purchase fresh, healthy products. The system facilitates
product management, order processing, and real-time inventory tracking.

TECHNOLOGY STACK
----------------
Backend:
  - Java Spring Boot (Microservices Architecture)
  - MySQL Database
  - WebSocket for real-time updates
  - RESTful APIs

Frontend:
  - Angular 18
  - TypeScript
  - Bootstrap 5
  - RxJS for reactive programming
  - Font Awesome icons

Microservices:
  1. User Service - User authentication and management
  2. Product Service - Product catalog and inventory
  3. Order Service - Order processing and tracking

4. DELIVERY SCHEDULE FEATURE
   -------------------------
   This feature allows vendors to specify when products will be delivered,
   helping customers plan their purchases accordingly.

   Vendor Side:
   - Add delivery schedule when creating/editing products
   - Choose from Saturday or Sunday delivery days
   - Optional field - can be left blank for no specific schedule
   - Located in product form after stock quantity field

   Customer Side:
   - Delivery schedule badge displayed on product cards
   - Shows delivery day (e.g., "Saturday Delivery" or "Sunday Delivery")
   - Displays calculated next delivery date
   - Uses calendar icon for visual clarity
   - Only shown when product has a delivery schedule

   Technical Implementation:
   - Backend: deliverySchedule field added to Product entity (String)
   - Frontend Model: deliverySchedule added to Product interface
   - Helper method: getNextDeliveryDate(schedule: string): Date
     * Calculates next Saturday (day 6) or Sunday (day 0)
     * Returns Date object for next occurrence
   - Display helper: getDeliveryScheduleDisplayText(schedule: string): string
     * Formats schedule text for display (e.g., "SATURDAY" -> "Saturday")

DATABASE SCHEMA
---------------
Key Entities:
- User (id, firstName, lastName, email, password, role, vendorId, customerId)
- Product (id, name, description, price, category, stockQuantity, vendorId,
           imageUrl, unitQuantity, productUnit, deliverySchedule)
- Order (id, orderName, orderQuantity, orderPrice, orderStatus, productId,
         vendorId, customerId, orderPlacedDate, orderDeliveredDate)

API ENDPOINTS
-------------
User Service:
- POST /api/users/register - User registration
- POST /api/users/login - User authentication

Product Service:
- GET /api/products - Get all products
- GET /api/products/vendor/{vendorId} - Get products by vendor
- POST /api/products - Create new product
- PUT /api/products/{id} - Update product
- DELETE /api/products/{id} - Delete product

Order Service:
- POST /api/orders - Create new order
- GET /api/orders/vendor/{vendorId}/active - Get active orders for vendor
- GET /api/orders/vendor/{vendorId}/history - Get order history for vendor
- PUT /api/orders/{id}/status - Update order status

VALIDATION RULES
----------------
Product Form:
- Name: Required, 2-100 characters
- Description: Required, 10-500 characters
- Price: Required, minimum 0.01
- Unit Quantity: Required, minimum 0.01
- Product Unit: Required
- Category: Required
- Stock Quantity: Required, minimum 0
- Image URL: Optional, must be valid URL format
- Delivery Schedule: Optional, either "SATURDAY" or "SUNDAY"

Order/Checkout Form:
- Delivery Address: Required, minimum 10 characters
- Contact Number: Required, must be 10 digits
